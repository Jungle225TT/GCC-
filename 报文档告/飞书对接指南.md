# 飞书多维表格对接指南

## 整体架构

```
每日定时运行
     │
     ▼
gcc_thinktank_scraper_v2.py    ← 抓取 + AI翻译 + AI简报
     │ 输出 JSON
     ▼
feishu_sync.py                 ← 读取JSON → 增量推送到飞书
     │ API调用
     ▼
飞书多维表格                    ← 团队在线查看、筛选、评论
```

## 第一步：创建飞书应用（5分钟，一次性）

1. 打开 https://open.feishu.cn → **创建企业自建应用**
2. 填写应用名称（如"GCC智库监测"）和描述
3. 进入应用 → **权限管理** → 搜索并开启：
   - `bitable:app` — 多维表格所有权限
4. 进入 **版本管理与发布** → 创建版本 → 提交发布
5. 记录 **App ID** 和 **App Secret**（在凭证与基础信息页）

## 第二步：创建多维表格（2分钟，一次性）

1. 在飞书中新建一个多维表格，命名如"GCC智库研究动态"
2. 从URL中提取两个关键参数：
   ```
   https://你的企业.feishu.cn/base/shcnXXXXXXXXX?table=tblYYYYYYYYY
                                      ↑ app_token          ↑ table_id
   ```
3. 在表格右上角 **...** → **添加协作者** → 搜索你的应用名 → 添加并赋予编辑权限

## 第三步：配置环境变量

```bash
# 添加到 ~/.zshrc（Mac）或 ~/.bashrc（Linux）
export FEISHU_APP_ID="cli_xxxxxxxxxxxxx"
export FEISHU_APP_SECRET="xxxxxxxxxxxxxxxxxxxxxxxx"
export FEISHU_APP_TOKEN="shcnXXXXXXXXX"
export FEISHU_TABLE_ID="tblYYYYYYYYY"

# 生效
source ~/.zshrc
```

## 第四步：运行同步

```bash
# 先抓取
python gcc_thinktank_scraper_v2.py --ai --playwright

# 再同步到飞书（自动找最新JSON）
python feishu_sync.py

# 或指定文件
python feishu_sync.py output/gcc_research_20260414_1035.json
```

首次运行会自动在多维表格中创建所需字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 发布日期 | 文本 | 文章发布日期 |
| 平台 | 文本 | 智库名称 |
| 标题 | 文本 | 英文原标题 |
| 中文标题 | 文本 | AI翻译的中文标题 |
| 链接 | 超链接 | 可直接点击打开原文 |
| 优先级 | 单选 | ⭐优先阅读 / 📄常规 / 📋简讯 |
| 内容类型 | 单选 | high / medium / low / unknown |
| 数据来源 | 单选 | RSS / HTML |
| 国家 | 文本 | 智库所在国 |
| 摘要 | 文本 | 文章摘要片段 |
| 抓取时间 | 文本 | 本次抓取的时间戳 |

## 第五步：设置定时自动运行

### Mac（crontab）

```bash
# 每天早上8点自动抓取并推送
crontab -e
# 添加：
0 8 * * * cd /你的路径/GccScraper && /opt/anaconda3/bin/python gcc_thinktank_scraper_v2.py --ai --playwright && /opt/anaconda3/bin/python feishu_sync.py >> output/cron.log 2>&1
```

### 或用 GitHub Actions（推荐，免费）

```yaml
# .github/workflows/gcc_scraper.yml
name: GCC Daily Scrape
on:
  schedule:
    - cron: '0 0 * * *'  # UTC 00:00 = 北京时间 08:00
  workflow_dispatch:      # 支持手动触发

jobs:
  scrape:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'
      - name: Install dependencies
        run: |
          pip install requests beautifulsoup4 feedparser anthropic playwright
          playwright install chromium
      - name: Run scraper
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: python gcc_thinktank_scraper_v2.py --ai --playwright
      - name: Sync to Feishu
        env:
          FEISHU_APP_ID: ${{ secrets.FEISHU_APP_ID }}
          FEISHU_APP_SECRET: ${{ secrets.FEISHU_APP_SECRET }}
          FEISHU_APP_TOKEN: ${{ secrets.FEISHU_APP_TOKEN }}
          FEISHU_TABLE_ID: ${{ secrets.FEISHU_TABLE_ID }}
        run: python feishu_sync.py
```

## 增量更新机制

每次运行 `feishu_sync.py`，会先读取表格中已有的所有文章URL，与新抓取的文章做对比：
- **新URL** → 写入表格
- **已存在的URL** → 跳过

所以可以放心反复运行，不会产生重复数据。

## 常见问题

**Q: 报错 "No permission to operate bitable"**
A: 需要在多维表格中将应用添加为协作者（第二步第3点）

**Q: 报错 "Invalid app_token"**
A: 检查 FEISHU_APP_TOKEN 是否是URL中 `/base/` 后面的那段（以 shcn 或 sh 开头）

**Q: 字段类型不匹配**
A: 首次运行会自动创建字段。如果手动创建了同名但不同类型的字段，需要删除后让脚本重新创建

**Q: 每次都是全量写入？**
A: 不是，脚本自动做URL去重，只写入新增文章
